## Why Definitions Matter

Consistent terminology:
- Reduces confusion in communication
- Helps AI developers understand context
- Makes documentation clearer
- Aligns team on expectations

## Quick Reference

| Term | One-Line Definition |
|------|-------------------|
| PR | Container for code changes, submitted for review before merge |
| Defect | Bug in existing functionality that needs fixing |
| Feature | New capability being added |
| Branch | Isolated workspace for making changes |
| Merge | Combining changes from one branch into another |
| Review | Examination of code changes before approval |

For full definitions, see [glossary.md](./glossary.md).
