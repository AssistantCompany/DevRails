# Contributing to DevRails

Thank you for your interest in contributing to DevRails!

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or request features
- Check existing issues before creating a new one
- Include clear steps to reproduce bugs
- For feature requests, explain the use case

### Submitting Changes

1. **Fork** the repository
2. **Create a branch** for your changes
   - `feature/your-feature-name` for new features
   - `bugfix/issue-description` for bug fixes
3. **Make your changes** following the existing code style
4. **Test** your changes
5. **Submit a Pull Request**

### Pull Request Guidelines

- Keep PRs focused on a single change
- Write clear commit messages using conventional format:
  - `feat: add new feature`
  - `fix: resolve bug`
  - `docs: update documentation`
- Update documentation if needed
- Reference related issues with `Fixes #123`

### Code Style

- Use clear, descriptive names
- Follow existing patterns in the codebase
- Comment complex logic
- Keep files focused and organized

### Documentation

- Update README.md for user-facing changes
- Update COMMANDS.md for new commands
- Update SETUP.md for installation changes
- Keep examples up to date

## Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/devrails.git

# Create a branch
git checkout -b feature/your-feature

# Make changes and test

# Submit PR
git push origin feature/your-feature
```

## Questions?

Open an issue with the "question" label if you need help.
